CHECKPOINT

Название

CHECKPOINT -- записать контрольную точку в журнал транзакций

Синтаксис

CHECKPOINT

Описание

Контрольная точка — это состояние в журнале транзакций, в котором все файлы данных содержат актуальные записи, в соответствии с информацией в журнале. При этом все файлы данных сохраняются на диск. Более подробно о том, что происходит во время контрольной точки, можно узнать в Разделе 29.4.

Команда CHECKPOINT приводит к принудительному выполнению контрольной точки в момент вызова, не дожидаясь периодической контрольной точки, производимой системой по графику (это настраивается параметрами в Подраздел 18.5.2). Команда CHECKPOINT не предназначена для применения при обычном использовании базы данных.

Если CHECKPOINT выполняется в процессе восстановления, вместо записи новой контрольной точки производится точка перезапуска (см. Раздел 29.4).

Выполнять CHECKPOINT могут только суперпользователи.

Совместимость

Команда CHECKPOINT является языковым расширением PostgreSQL.
